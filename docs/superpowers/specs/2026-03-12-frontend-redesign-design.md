# Frontend Redesign — Design Spec

## Overview

Redesign the frontend UI for a voice-first AI accountability partner app. The app helps users track up to 3 habits via real-time voice check-ins. The UI should feel natural, intuitive, and frictionless — not a clinical dashboard or a generic AI wrapper. Function over beauty. The voice experience is the product.

**Stack:** Next.js 15, React 19, TypeScript, Tailwind CSS 4, shadcn/ui (Radix + Tailwind)

**Default mode:** Dark. Light mode available via system preference (follows OS setting, no in-app toggle for hackathon).

**Target:** Mobile-first. Primary use case is phone. Desktop is functional but not optimized.

## Schema Changes Required

These changes to existing data models are needed to support this design:

- `UserProfile`: add `birthday: string` field
- `CheckInSession`: rename `microCommitment: string` → `commitments: string[]` (array of commitments per session)
- `CheckInSession`: add `durationSeconds: number`
- `CheckInSession`: add `habitsCovered: string[]` (habit IDs discussed in session)
- `CheckInSession`: add `insight: string` (forward-looking insight mirroring user's words, generated by agent)
- `UserProfile.displayName`: populated from onboarding voice conversation (overrides auth provider value)
- `patternsFlagged`: still stored in Firestore (agent uses it internally) but not displayed in the UI. Patterns are addressed in the voice conversation only.

### Backend Tool Updates

The `save_session_summary` tool in `backend/agent/tools.py` must be updated to match:
- `micro_commitment: str` → `commitments: list[str]`
- Add `insight: str` parameter
- Add `habits_covered: list[str]` parameter
- `durationSeconds` is computed server-side from WebSocket connection open/close timestamps and saved automatically — not passed through the agent tool.

The `adk-system-prompt.md` closing ritual section should be updated to reflect the new tool signature (multiple commitments, insight generation).

### Persona Shifts

Persona affects greeting tone, message border, and CTA shade — not layout. All three use the same structure.

| Element | Coach | Friend | Reflective |
|---------|-------|--------|------------|
| Greeting | "Let's go, Jordan." | "Hey Jordan" | "Welcome back, Jordan" |
| Message border | Sage border-left | None | None |
| Dark CTA | `#6ea88e` (sage) | `#82b89a` (bright sage) | `#b0b8c4` (silver) |
| Light CTA | `#3a4048` (charcoal) | `#5a7a68` (sage) | `#3a4048` (charcoal) |

### Typography

- `heading-lg`: 26px, weight 500, letter-spacing -0.4px (home greeting)

- Card padding: 14px
- Section gap: 16px (between cards), 20–24px (between sections)

### General Rules

- No emojis anywhere in the UI
- Streaks displayed as "X days" (plain text)
- Insights always mirror the user's own words from conversations, forward-looking
- Missed commitments are addressed in the voice conversation, never as callouts in the UI

## Screens

### 1. Home (Landing)

Conversation-first — not a dashboard. Opens like a message from your partner.

**Structure:**
- Time-of-day label (top, muted, 12px)
- Greeting (persona-dependent, 26px weight 500)
- Message bubble (white card in light, `#2e3440` panel in dark, one-line message)
  - Coach variant: sage border-left on message
- Two buttons: "Check in" (primary CTA) | "Schedule" (ghost/outline)
- Streak row at bottom (e.g., "6 days · 1 photo today", muted text, no emojis)

**Message content:** One line, context-aware. Examples: "Six days. Don't stop now." / "Photo logged — wanna chat?" / "You've been consistent. Ready?"

**"Schedule" button:** Deferred for hackathon. Button is visible but v2 scope. For the demo, tapping it can show a simple "Coming soon" toast or open a basic time picker that saves to `UserProfile.dailyCheckInTime`.

### 2. Check-in Voice Session

The core experience. Habit progress pills + live transcript. The screen reflects the conversation's structure.

**Structure:**
- Top bar: timer (left) | partner name (center) | end button (right, red square in circle)
- Habit progress pills: shows which habits have been covered (✓), which is active (→), which are pending (dimmed). Updates in real time as the conversation progresses.
- Waveform: centered, sage-colored bars, animated while partner or user is speaking
- Speaking indicator: "Max is speaking..." / "Listening..." below waveform
- Live transcript: bottom panel (`#2e3440`), italic, shows the current spoken line
- Bottom controls: camera button (left) | mute toggle (center, large) | overflow menu (right)

**Camera button:** Opens camera for mid-session photo sharing (sent to the AI for vision analysis).

**No context cards or commitment captures during session** — those surface in the debrief after.

### 3. Session End (Debrief)

Rendered inline within the `/check-in` page — not a separate route. When the WebSocket session ends, the check-in page transitions from the voice UI to the debrief view. Session data (commitments, habits covered, insight, streaks) comes from the `save_session_summary` tool call the agent makes at conversation end. If the user navigates away before the debrief loads, the data is still saved to Firestore and viewable from History.

**Structure:**
- Header: milestone label if earned ("NEW RECORD" in sage, 12px) or "SESSION COMPLETE"
  - Milestone: "7 days on Betting." (22px headline)
  - No milestone: "Nice work, Jordan." (22px headline)
  - Sub-line: duration + habits covered count
- Habit pills: same style as check-in (✓ for covered, — for skipped)
- Commitments card: "YOUR COMMITMENTS" label, bullet list of commitments extracted from conversation
- Insight card: "INSIGHT" label, 1–2 sentences that reinforce the user's own realization from the conversation
  - Style: mirrors their words back. "You said it yourself — when you eat well, you don't feel the need to bet. Hold onto that this weekend."
  - Always forward-looking, never a callout or judgment
  - Specific to what was discussed in THIS session
- Streak row: three compact cards showing current streak per habit ("4 days", "7 days", "2 days")
  - Milestone streak gets sage-highlighted border
- "Done" button returns to home

### 4. History

Timeline of past sessions. Simple list, tap to expand.

**Structure:**
- Header: "History" (22px) | session count (right, muted)
- Grouped by week: "THIS WEEK", "LAST WEEK", etc. (11px, muted, uppercase). Weeks with no sessions are skipped. Show up to 8 weeks, then paginate or lazy-load.
- Session cards: each is a `#2e3440` panel containing:
  - Row 1: day name (left, 13px white) | duration (right, 12px muted)
  - Row 2: habit pills (small, 10px, same sage style)
  - Row 3: one-line insight from that session (12px, secondary text)
- Tap a card to expand into the full debrief (commitments, streaks at that point, full insight)
- Scrollable list

### 5. Weekly Summary

Report card format. Habits expandable for per-habit insights.

**Structure:**
- Header: "WEEK OF MAR 3 – 9" (11px muted) | "Your week" (22px)
- Stats row: three compact cards — check-in count, total minutes, commitments kept
- Per-habit cards (collapsed by default):
  - Row: habit name (left, 13px) | streak (right, sage)
  - 7-segment progress bar: filled segments = active days (sage), empty = `rgba(255,255,255,0.06)`
  - Tap to expand: reveals milestone label (if earned), conversation-sourced insight for that habit, and commitments made for that habit during the week
  - Expanded card gets sage border
  - Tap again to collapse
- "Done" button

### 6. Onboarding

Single voice conversation that collects everything. Progressive field fill on screen.

**Flow:**
1. User signs up (email/password — existing auth)
2. Voice onboarding begins immediately
3. AI introduces itself conversationally, asks their name, birthday
4. Transitions naturally into partner setup: "What do you want to call me?", persona preference, habits (up to 3), check-in time (asked near end)
5. Language auto-detected from speech (Gemini detects the language from the audio stream; default fallback is English; user can override in review)
6. Review screen appears when conversation ends

**During voice (progressive fill):**
- "Setting up" headline
- Fields appear and fill as the AI captures answers
- Completed fields: solid panel with value
- Active field: sage border
- Upcoming fields: dimmed, placeholder dash
- Waveform + transcript at bottom (smaller than check-in — form is the focus)

**Review screen:**
- "Look right?" headline + "Tap anything to edit" subtitle
- Personal info (name, birthday) as a compact row at top — lighter text (`#b0b8c4`), smaller cards, no "edit" label, just tappable. Visually quieter than partner setup.
- Thin divider
- Partner setup fields below (partner name, style, habits, check-in time) — full-size cards with "edit" affordance
- Tapping "edit" expands inline: text input for name, tappable pills for style (Coach / Friend / Reflective), pill editor for habits, time picker for check-in
- "Looks good" button confirms and navigates to home

## Navigation

- Home is the landing screen (not a dashboard)
- Minimal header on home: partner name (left), two icon buttons (right) — clock icon for History, bar-chart icon for Weekly Summary
- Settings: reuses the onboarding review screen UI. Accessible from an overflow/gear icon in the home header or History/Summary screens. Same inline edit behavior.
- No bottom tab bar. Keep navigation minimal for hackathon.

## Technical Notes

### Voice Architecture

All voice sessions (both check-in and onboarding) use the server-mediated WebSocket architecture: browser ↔ FastAPI ↔ Gemini Live API. The existing client-side `@google/genai` SDK usage in onboarding should be replaced with the same WebSocket approach used by check-in. This keeps API keys server-side and enables tool calls.

### Video Removal

The existing check-in page streams continuous video at 1fps via WebSocket. The redesign removes this in favor of audio-only with on-demand photo capture (camera button). The `type: 'video'` WebSocket message type is no longer sent continuously. Photo capture sends a single frame when the user taps the camera button.

### Auth Screen

The existing login/signup UI (`page.tsx` unauthenticated state) should be restyled to match the Dusk + Sage palette. Same functionality (Google sign-in + email/password), new colors. Not a priority for hackathon — functional is fine.

### Error States

- **Microphone denied:** Show a centered card with "Microphone access needed" + a "Try again" button that re-requests permission.
- **WebSocket connection failed:** "Couldn't connect. Check your connection." + "Retry" button.
- **Mid-session disconnect:** "Connection lost. Reconnecting..." with auto-retry (3 attempts). If all fail, show debrief with whatever data was captured.

## Design Principles

1. **Function over beauty.** The voice experience is what wows. UI supports it, doesn't compete.
2. **Not a wrapper.** The UI should reflect that the agent is thinking, remembering, and tracking — not just piping audio.
3. **No callouts.** Missed commitments and negative patterns are addressed in the voice conversation, not surfaced as judgments in the UI.
4. **Mirror, don't instruct.** Insights reinforce the user's own words and realizations.
5. **Conversation-first.** The app opens like a message, not a control panel.
6. **No emojis.** Clean text throughout.
